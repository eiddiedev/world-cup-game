# 抖音互动空间 8MB Demo 设计

## 目标

从完整版生成一个独立横屏 Demo，在不改变完整版构建和素材的前提下，将未压缩产物控制在 8 MiB 内，并保证 `index.html` 只使用相对路径。

## 范围

- 可玩球队保留法国和库拉索，形成强队与弱队的明显体验差异。
- 保留完整流程：球队选择、球员招募、排兵布阵、小组赛、淘汰赛、Phaser 比赛、决策、换人、点球、赛后统计和结局。
- 保留两队原始球员图片，不对原图做有损压缩。
- 保留全部赛事国旗，避免随机淘汰赛对手缺图。
- 保留属性、比赛事件、金币、锁、庆祝和主 Logo 等功能资源。
- Demo 首页不加载大背景和第二帧 Logo；阵容球场使用最近邻缩放的轻量副本。
- 中文像素字体按项目实际字符生成子集。

## 架构

- Vite 使用 `douyin` mode 输出到 `dist-douyin/`，设置 `base: './'` 且关闭默认 `public/` 全量复制。
- `scripts/build-douyin-demo.mjs` 负责构建、白名单复制、字体子集、轻量球场生成、相对路径检查、8 MiB 门禁和 ZIP 打包。
- `src/config/runtime.js` 集中定义 Demo 模式、可玩球队和独立存档键。
- 完整版继续使用 `npm run build`，Demo 使用 `npm run build:demo`。

## 验收

- `dist-douyin/index.html` 可通过静态服务器从任意子目录加载。
- 产物中不存在以 `/assets/` 开头的资源引用。
- 未压缩目录与 ZIP 均不超过 8 MiB。
- 球队选择只显示法国和库拉索。
- 完整版构建仍显示十支球队并通过现有测试。
