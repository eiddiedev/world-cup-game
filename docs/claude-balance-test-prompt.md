## 核心测试目标

建立一个可重复运行的 Monte Carlo 模拟，至少覆盖以下球队：

- 法国：强队。20 次完整世界杯模拟中，理想夺冠次数约 15 次。可接受区间先设为 12-17 次。
- 巴西/阿根廷/葡萄牙/德国：强队或准强队，夺冠率应明显高于中弱队，但低于或接近法国。
- 日本/摩洛哥/挪威：中档挑战队，应该有出线和偶尔深轮次机会，但不应稳定夺冠。
- 新西兰/库拉索：弱队。库拉索 20 次模拟理想夺冠 0 次；100 次模拟约 0-2 次夺冠，目标接近 1 次。

完整世界杯模拟定义：

1. 自动为球队选择阵容。
2. 自动排出 11 人首发。
3. 自动模拟 3 场小组赛。
4. 若出线，自动模拟 16 强、8 强、4 强、决赛。
5. 输出每次 run 的最终阶段：小组出局、16 强、8 强、4 强、亚军、冠军。

## 自动选人规则

为了减少人工操作偏差，请实现 3 种选人策略：

1. `strongest`: 在预算内优先选高评分球员，目标 13 人左右。
2. `balanced`: 位置完整、预算合理，目标 15 人左右。
3. `cheap`: 优先低价凑人数，目标 18 人左右。

每种策略都必须保证：

- 至少 1 名 GK。
- 首发 11 人位置合法。
- 如果预算不足或位置不足，要在报告中标记，不要静默跳过。

## 输出指标

每个球队至少输出：

- 模拟次数。
- 小组出线率。
- 16 强、8 强、4 强、决赛、冠军比例。
- 场均进球、场均失球。
- 决策触发次数均值。
- 点球大战次数。
- 预算使用率。
- 平均购买人数。

输出格式：

1. 终端打印一份 Markdown 表格。
2. 写入 `docs/balance/YYYY-MM-DD-balance-report.md`。
3. 如果新增脚本，写清楚运行命令。

## 运行规模

第一阶段快速测试：

```bash
npm test -- --run
node scripts/balance-sim.mjs --teams france,curacao --runs 20 --strategy balanced
```

第二阶段长跑测试：

```bash
node scripts/balance-sim.mjs --teams france,brazil,argentina,portugal,germany,japan,morocco,norway,newzealand,curacao --runs 100 --strategy balanced
```

第三阶段策略交叉测试：

```bash
node scripts/balance-sim.mjs --teams france,curacao --runs 300 --strategy strongest
node scripts/balance-sim.mjs --teams france,curacao --runs 300 --strategy balanced
node scripts/balance-sim.mjs --teams france,curacao --runs 300 --strategy cheap
```

如果运行很久，不要停止。持续跑到得到稳定样本后再总结。

## 判断标准

优先关注这些问题：

1. 强弱队差距是否明显。
2. 法国是否太弱或太强。
3. 库拉索是否异常夺冠过多。
4. 小组赛是否过于随机。
5. 淘汰赛是否出现弱队连赢强队太频繁。
6. 金卡隐藏属性是否过强。
7. 预算是否真的迫使玩家取舍。

## 修改建议规则

如果发现不平衡，先只写建议，不要立刻大改。

可以建议微调的范围：

- 球队/球员 rating、price。
- 决策成功率公式中的小系数。
- 淘汰赛对手强度模拟。
- 金卡隐藏属性倍率。
- 弱队 upset 概率。

不要建议：

- 重做 UI。
- 重做整个比赛系统。
- 引入大型新库。
- 改成真实足球经理复杂模拟。
